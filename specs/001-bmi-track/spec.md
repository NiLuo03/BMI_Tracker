## 一、项目概述

### 1.1 项目目标
开发一款**单机桌面应用**（非网页、非移动端），帮助用户记录 BMI 数据、查看变化趋势、预测未来 BMI 值，并基于用户体质通过 AI 生成膳食推荐。

### 1.2 运行环境
- **操作系统**：Windows （JDK 17+）
- **数据库**：MySQL 8.0（安装在一台电脑上，局域网内其他两台可连接）
- **界面框架**：JavaFX + SceneBuilder（FXML 文件描述布局）

## 二、功能需求（共 6 大模块）

### 模块 1：用户管理

| 功能点 | 输入 | 输出 / 行为 | 异常处理 |
|--------|------|-------------|----------|
| **注册** | 账号(userName)、密码(password)、年龄(userAge)、性别(sex: 0男/1女) | 注册成功 → 跳转登录页；注册失败 → 弹窗提示错误原因 | 账号已存在 → 提示“账号已被占用”；任何字段为空 → 提示“请完整填写” |
| **登录** | 账号、密码 | 验证通过 → 进入主界面；失败 → 弹窗提示 | 账号不存在或密码错误 → 统一提示“账号或密码错误”（防撞库） |
| **密码修改** | 旧密码、新密码(两次) | 修改成功 → 提示并跳转登录页；失败 → 弹窗 | 旧密码错误 → 提示“原密码不正确”；两次新密码不一致 → 提示“两次输入不同” |
| **个人信息查看/编辑** | 展示：userId, userName, userAge, sex, createTime | 允许修改身高、体重、年龄（密码单独改） | 身高/体重须为正数，否则提示“请输入有效数值” |

> **字段说明**：`userId` 自增主键；`createTime` 由数据库自动生成（`CURRENT_TIMESTAMP`）。

### 模块 2：BMI 计算与存储

| 功能点 | 输入 | 输出 / 行为 | 算法 / 规则 |
|--------|------|-------------|-------------|
| **计算 BMI** | 身高(height, cm)、体重(weight, kg) | 输出 BMI 值（保留一位小数） | 公式：`BMI = weight / (height/100)^2` |
| **自动匹配健康区间** | BMI 数值 | 输出区间名称（见右表） | `<18.5`→"偏瘦"；`18.5~24`→"正常"；`24~28`→"超重"；`≥28`→"肥胖" |
| **存储记录** | userId, height, weight, bmi, 计算时间 | 写入 `bmi_records` 表 | 每条记录自动生成 `recordId` 和 `createTime` |

> **交互逻辑**：用户在主界面输入身高体重，点击“记录并计算”按钮 → 同时完成计算、匹配、存储三步，并弹窗显示“您的BMI为 X.X，状态：正常”。

### 模块 3：历史记录与折线图

| 功能点 | 输入 | 输出 / 行为 | 数据要求 |
|--------|------|-------------|----------|
| **历史数据列表** | 当前登录 userId | 按时间**倒序**展示所有历史记录（含：日期、身高、体重、BMI、健康区间） | 表格控件（TableView）展示，每页显示10条（若超过） |
| **BMI折线图** | 当前登录 userId | 以“日期”为 X 轴，“BMI值”为 Y 轴绘制折线图 | 仅当记录 ≥ 2 条时才显示折线；若 < 2 条，显示提示“暂无足够数据绘制折线图” |

> **注意**：折线图使用 JavaFX 内置的 `LineChart` 控件，数据直接从 `bmi_records` 表按时间升序取出。

### 模块 4：趋势预测

> **触发条件**：当前用户的历史 BMI 记录 **≥ 4 条** 时，“趋势预测”按钮才可点击；若 < 4 条，按钮置灰，鼠标悬停显示“至少录入4次数据，趋势预测更准确哦~”。

#### 4.1 核心算法：最小二乘法线性回归
- **数学模型**：`BMI = a × (天数序号) + b`
- **计算步骤**（伪代码）：
  ```
  1. 查询该用户所有历史记录，按时间升序排列，得到 n 条 (x_i, y_i)
  2. x_i = 0, 1, 2, ..., n-1（天数序号）；y_i = 对应的 BMI 值
  3. 计算 x 的平均值 x̄，y 的平均值 ȳ
  4. 计算斜率 a = Σ[(x_i - x̄)(y_i - ȳ)] / Σ[(x_i - x̄)²]
  5. 计算截距 b = ȳ - a × x̄
  6. 预测 7 天后：x_pred = n + 6（因为第 n 条记录对应 x=n-1，7天后即 x=n+6）
  7. 预测值 = a × x_pred + b
  ```
- **输出要求**：在界面显示完整的回归方程 `BMI = a×天数 + b`，保留两位小数。

#### 4.2 四个增强组件

| 增配包 | 实现要求 |
|--------|----------|
| **① 回归方程展示** | 在折线图下方用文字显示公式，并翻译斜率含义，如：“您的BMI平均每天上升/下降 0.12” |
| **② 健康区间预警带** | 在折线图背景中，用半透明色块标识：绿色带（18.5~24）、黄色带（<18.5 或 24~28）、红色带（≥28）。将预测的 7 天后 BMI 值用**闪烁红点**标在图上；若落在红色/黄色区，下方文字变红加粗：“⚠️ 预警：7天后将进入 [超重/肥胖] 区间！” |
| **③ 模拟“控制线”对比** | 在图上画两条线：**真实历史+预测线**（实线+虚线）和 **理想平缓线**（从当前 BMI 值水平延伸的平直线）。两条线对比展示 |
| **④ 历史极值标注** | 在折线图上用**红色圆点**标出历史最高 BMI（附带日期），用**绿色圆点**标出历史最低 BMI（附带日期） |

### 模块 5：AI 膳食推荐（调用 Coze API）

#### 5.1 实现方案
- 不在 JavaFX 内实现 AI 算法，而是**调用 Coze 平台发布的智能体 API**。
- **智能体设计要点**（在 Coze 平台配置）：
  - **人设**：“你是一位资深注册营养师，擅长根据用户体质数据推荐一日三餐。”
  - **输入参数**：用户年龄、性别、身高、体重、BMI、健康区间、偏好标签（如“不吃辣”“少碳水”）
  - **输出格式**：`{ "breakfast": "全麦面包2片+牛奶250ml (450kcal)", "lunch": "...", "dinner": "...", "totalCal": "1850kcal" }`
  - **知识库**（可选）：上传常见食物营养数据，供 AI 引用

#### 5.2 JavaFX 端调用要求
- **输入**：当前用户的所有体质数据 + 偏好（从数据库读取）
- **请求方式**：HTTP POST（JSON 格式）
- **超时设置**：5 秒（Coze API 若超时，显示“AI 服务繁忙，请稍后再试”）
- **输出展示**：将返回的早餐、午餐、晚餐、总热量用卡片形式美观展示在界面上

> **用户偏好识别**：在“个人信息编辑”页面增加多选框（如“不吃辣”、“不吃海鲜”、“低卡偏好”等），存入数据库（可用 `preferences` 字段存储 JSON 或逗号分隔字符串）。

### 模块 6：食物选择参考

#### 6.1 数据来源
手动录入至少 **50 种常见食物** 到 `foods` 表，包含字段：`foodId, foodName, category(主食/肉类/蔬菜/水果/饮品), calories(每100g大卡), protein, fat, carb`。

#### 6.2 功能点

| 功能点 | 输入 | 输出 |
|--------|------|------|
| **食物对比** | 用户选择 2~3 种食物 | 以表格形式横向对比：热量、蛋白质、脂肪、碳水，并自动高亮“热量最低”和“蛋白质最高”的单元格 |
| **食物榜单** | 下拉选择分类（如“蔬菜”） | 按热量从低到高排序，展示前 10 名，并显示“低卡推荐”徽章 |

## 三、数据库设计（共 4 张表）

### 表 1：`users`（用户表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| userId | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| userName | VARCHAR(20) | UNIQUE, NOT NULL | 登录账号 |
| password | VARCHAR(64) | NOT NULL | 密码（明文存储即可，实训作业无需加密） |
| userAge | INT | NOT NULL | 年龄 |
| sex | TINYINT | NOT NULL, DEFAULT 0 | 0=男, 1=女 |
| height | DECIMAL(5,2) | | 身高(cm) |
| weight | DECIMAL(5,2) | | 体重(kg) |
| preferences | VARCHAR(200) | | 偏好标签，逗号分隔（如“不吃辣,低卡”） |
| createTime | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

### 表 2：`bmi_records`（BMI 记录表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| recordId | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| userId | INT | FOREIGN KEY → users.userId | 对应用户 |
| height | DECIMAL(5,2) | NOT NULL | 记录时的身高 |
| weight | DECIMAL(5,2) | NOT NULL | 记录时的体重 |
| bmi | DECIMAL(4,1) | NOT NULL | 计算出的 BMI 值 |

| createTime | DATETIME | DEFAULT CURRENT_TIMESTAMP | 记录时间 |

### 表 3：`foods`（食物表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|

| calories | DECIMAL(6,1) | NOT NULL | 每100g热量(大卡) |
| protein | DECIMAL(5,1) | | 每100g蛋白质(g) |
| fat | DECIMAL(5,1) | | 每100g脂肪(g) |
| carb | DECIMAL(5,1) | | 每100g碳水(g) |

### 表 4：`recommendations`（膳食推荐记录表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| recId | INT | PRIMARY KEY, AUTO_INCREMENT | 推荐ID |

| breakfast | VARCHAR(100) | | 早餐内容 |
| lunch | VARCHAR(100) | | 午餐内容 |
| dinner | VARCHAR(100) | | 晚餐内容 |

## 四、非功能性需求

- **响应时间**：无严格并发要求（单机使用），界面切换和按钮点击响应须在 **1 秒内** 完成
- **代码风格**：遵循 Java 命名规范，关键逻辑需加中文注释（方便老师查阅）
- **容错处理**：所有数据库操作（增删改查）必须包裹在 `try-catch` 中，异常时弹出友好中文提示（不显示堆栈信息）

## 五、界面布局约定（SceneBuilder 约束）

- 所有 FXML 文件存放在 `src/main/resources/fxml/` 目录
- 主界面采用 `BorderPane` 布局，左侧为导航菜单（含：首页/BMI记录/趋势预测/膳食推荐/食物参考），右侧为内容展示区
- 所有弹窗统一使用 `Stage` + `Modality.WINDOW_MODAL` 方式，确保模态（必须关闭弹窗才能操作主界面）
