## 🧩 C 的 3 种可行定位

### 选项 C1：专职 Controller 逻辑
C 负责所有 FXML 对应的 Controller 类——即：A 拖完界面后，C 在 Controller 里写按钮点击事件、调用你的 Service 方法、填充 TableView/LineChart。

**C 的前期工作（本周）**：
- 注册/登录 Controller：调用 `UserService.register()` 和 `login()`
- BMI 计算 Controller：调用 `BmiService.calculate()` 并调用你的 `saveBmiRecord()`
- 注意：C 写 Controller 时，需要你提前定义好 Service 接口（方法名、参数类型、返回值），你们俩同时并行开发

### 选项 C2：专职算法 + AI 集成
C 专门负责 **M9（最小二乘法类）、M10（方程展示）、M11（预警带计算）、M12（控制线数据生成）、M13（极值标注数据准备）** 和 **M15（CozeClient）**。

**C 的前期工作（本周）**：
- 写 `LinearRegression.java`（纯算法类，不涉及界面，不涉及数据库）
- 写 `CozeClient.java`（纯 HTTP 调用工具，不涉及界面）
- 你只需要告诉 C：`getRecordsByUserId(userId)` 返回的数据结构是什么，他就能自己测

### 选项 C3：专职食物参考模块
C 只负责 M17（食物对比界面+逻辑）和 M18（食物榜单界面+逻辑），外加 M6/M7（历史列表和折线图）的简单展示逻辑。

**C 的前期工作（本周）**：
- 在你录入完 50 种食物后，写 `SELECT * FROM foods ORDER BY calories` 等简单查询
- 把查询结果填到 TableView 里
